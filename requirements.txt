# --------------------------
# Core: DeepEval Framework
# --------------------------
deepeval>=1.0.0              # Complete LLM evaluation framework
                             # Includes ALL of the following in one package:
                             #
                             # Test Case:
                             #   LLMTestCase          — evaluation unit (input, output, context)
                             #   LLMTestCaseParams    — enum (INPUT, ACTUAL_OUTPUT, EXPECTED_OUTPUT,
                             #                          RETRIEVAL_CONTEXT)
                             #
                             # Metrics (14+ built-in):
                             #   GEval                — custom LLM-as-judge for any criteria
                             #   FaithfulnessMetric   — RAG claim verification vs context
                             #   AnswerRelevancyMetric — does output answer the question?
                             #   HallucinationMetric  — claims contradicted by ground truth
                             #   ContextualPrecisionMetric — retrieved chunks relevance
                             #   ContextualRecallMetric   — all relevant info retrieved?
                             #   ContextualRelevancyMetric — overall context quality
                             #   ToxicityMetric       — built-in toxicity detection
                             #   BiasMetric           — gender, racial, political bias
                             #   ToolCorrectnessMetric — agent tool selection evaluation
                             #   TaskCompletionMetric — agent task completion evaluation
                             #
                             # Testing:
                             #   assert_test()        — pytest assertion for metrics
                             #   evaluate()           — batch evaluation across many test cases
                             #   deepeval test run    — CLI for running pytest-based evals
                             #
                             # Confident AI integration:
                             #   login_with_confident_api_key() — connect to dashboard

# --------------------------

# ============================================================
# NOTES FOR DEVELOPERS
# ============================================================
#
# 1. DEEPEVAL IS A COMPLETE PACKAGE
#    Just `pip install deepeval` installs everything you need for
#    this project. OpenAI is a dependency of deepeval itself.
#    You do NOT need to separately install openai for basic usage.
#
# 2. COST PER EVALUATION
#    Each metric.measure(test_case) call makes 1-2 LLM API calls.
#    Approximate costs with GPT-4o-mini as judge:
#      GEval:              ~$0.0001 per evaluation
#      FaithfulnessMetric: ~$0.0003 per evaluation (2 steps)
#    With 100 test cases and 3 metrics: ~$0.06 total (very cheap)
#    With 10,000 test cases:            ~$6.00 total
#    Use GPT-4o-mini as judge to minimize costs.
#
# 3. SETTING THE JUDGE MODEL
#    GEval:
#      metric = GEval(..., model="gpt-4o-mini")
#    FaithfulnessMetric:
#      metric = FaithfulnessMetric(threshold=0.7, model="gpt-4o-mini")
#    Set model explicitly to control costs and reproducibility.
#
# 4. GEVAL CRITERIA QUALITY
#    GEval score quality directly depends on criteria clarity:
#      BAD:  criteria="Is the output good?"
#      GOOD: criteria="Coherence - whether all sentences connect
#                       logically and the response flows naturally
#                       from introduction to conclusion"
#    Spend time writing precise, unambiguous criteria.
#
# 5. FAITHFULNESSMETRIC requires retrieval_context
#    If retrieval_context is not provided in LLMTestCase,
#    FaithfulnessMetric will raise a ValueError.
#    Always pass retrieval_context when testing RAG outputs.
#
# 6. THRESHOLD GUIDELINES BY APPLICATION TYPE
#    Creative/casual:    threshold=0.5  (lenient)
#    General chatbot:    threshold=0.7  (standard)
#    Enterprise/business: threshold=0.8  (strict)
#    Medical/legal/finance: threshold=0.9  (very strict)
#    Adjust per metric and per risk level of your application.
#
# 7. RUNNING WITH PYTEST
#    Create test_llm.py and run:
#      deepeval test run test_llm.py
#    NOT: pytest test_llm.py
#    The deepeval CLI applies additional setup for async metrics.
